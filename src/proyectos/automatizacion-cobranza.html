Automatización de Cobranza

Modernización del manejo de datos y de las campañas de cobranza de una empresa cuyo negocio entero dependía de un Excel masivo: de un archivo compartido en un servidor a un esquema SQL Server, campañas automatizadas y un dashboard de desempeño diario.

Agecob es una empresa dedicada a la cobranza judicial de casas comerciales. Cuando un cliente moroso supera los plazos de pago, el camino habitual es un juicio de embargo de bienes o propiedades. Pero ese camino no siempre es viable: hay deudas más baratas que el propio juicio, y clientes que no tienen ningún bien a su nombre.

Ahí entra la cobranza extrajudicial. Antes de iniciar el proceso legal, se advierte al cliente y se le ofrece repactar la deuda para paralizar el embargo. El modelo tiene una lógica tributaria concreta: las casas comerciales pagan impuestos por sus deudas, pero las deudas vencidas que logran recuperar quedan exentas. Por eso contratan empresas externas para recuperar ese dinero y pagan comisiones millonarias según el porcentaje recuperado. Agecob operaba una rama de cobranza extrajudicial con carteras entregadas por Falabella y Paris, gestionando la cobranza telefónica desde un CRM llamado NeoHotel.

El gran problema: toda la base de datos de la empresa era un Excel masivo guardado en un servidor. Sobre ese archivo, y según los resultados de cobranza de cada día, había que armar campañas masivas de correos — un proceso lento y manual. El trabajo consistió en modernizar tanto el almacenamiento como la operación de campañas.

Lo que se construyó

Migración a SQL Server

Se diseñó un esquema de base de datos en SQL Server para reemplazar el Excel masivo, dándole a la empresa un almacenamiento parejo, consultable y confiable.

Campañas automatizadas

Un programa en Python que, a partir de una lista de clientes, separa las campañas por ejecutiva y por categoría de cliente, y luego carga y prepara la campaña de correos de forma automática con Selenium — dejando solo el envío como último paso manual.

Dashboard de desempeño

Scraping automático del CRM NeoHotel que alimenta una carpeta en SharePoint, la cual a su vez nutre un dashboard de Power BI con el desempeño diario de cobranza y el recupero de cada campaña.

El resultado fue acortar drásticamente el ciclo de cobranza: la preparación de campañas —antes un trabajo manual sobre Excel— quedó reducida a un único clic de envío, con las listas ya segmentadas por ejecutiva y por categoría de cliente. Y por primera vez, la dirección tuvo visibilidad diaria del recupero real de cada cartera.

Stack & tecnologías
