Backend Proyecto principal Comercial Kuppel SPA

Kuppel Management

Plataforma web que extiende el ERP Softland Cloud para cubrir sus huecos de información: nació como reportería y creció hasta convertirse en el sistema híbrido de datos, control y ventas de la empresa.

En Comercial Kuppel SPA operamos con el ERP Softland en su versión Cloud. El sistema funciona bien para la operación, pero muestra la información de forma incómoda: sus dashboards e informes descargables no bastaban para tomar decisiones ni evaluar la situación de la empresa con agilidad.

El proyecto es la culminación de meses construyendo esos mismos análisis en Power BI. Una vez validado qué información necesitaba realmente el negocio y cómo debía presentarse, se migró la lógica de esos dashboards a una plataforma propia —ganando control total sobre los datos, la interacción y las reglas de negocio que una herramienta de BI no permitía.

Kuppel Management se construyó para cerrar esa brecha. Se conecta a la base de datos espejo SQL Server contratada con Softland y, desde ahí, entrega la información que el ERP no sabía presentar. Está desarrollado en Python + Django, desplegado como aplicación web en PythonAnywhere, con respaldo de base de datos y almacenamiento de adjuntos en Backblaze. La base de código sigue las metodologías MVC y DRY para mantenerse ordenada y mantenible a medida que la plataforma crecía.

Trabajar sobre una base de datos ajena y extensa tiene su propia fricción: hay que descubrir en qué tabla y columna vive cada dato antes de poder consultarlo. Para agilizar ese estudio me construí una utilidad interna en Python —una herramienta personal, no un producto— que escanea la BD buscando valores específicos por todas las tablas y devuelve dónde aparecen, lista tablas y columnas para preparar consultas, y permite cruzar dos campos en una misma fila. Nada que no se pueda hacer con queries, pero me ahorró incontables horas de exploración manual.

Lo que empezó como reportería terminó siendo una plataforma completa: no solo generaba reportes y dashboards, también sirvió como sistema de ingreso de datos y registros, manteniendo una coexistencia híbrida con el ERP. Finalmente se integró con el plugin de e-commerce —consumido mediante una API SOAP sobre XML— para crear una plataforma de ventas independiente, entregada a los vendedores, con reglas de negocio que Softland no era capaz de aplicar — mejorando el control y la trazabilidad de toda la empresa.

Destacados

La plataforma agrupa varios subsistemas. Estos son los de mayor impacto:

Conciliaciones bancarias

Paneles que comunican áreas para identificar qué pagos corresponden a cada movimiento bancario del día, agilizando la conciliación. Sobre ese registro se montó un dashboard con lo pendiente por conciliar y el estado de las conciliaciones, por periodo y por banco.

Plataforma de vendedores

Cada vendedor recibe su propio panel: rendimiento de ventas, clientes, productos más vendidos, trazabilidad, disponibilidad en bodega de compras pendientes, solicitud de compras y productos nuevos, y estado de sus clientes. Toda la información de venta en un solo lugar.

Auditoría automatizada

Paneles que detectan de forma automática actividades irregulares —ingreso de mercadería sin costo, sobrecompras de productos, estados anómalos— para gestionarlas sin depender de una observación manual y constante.

Además, la plataforma incluye un panel de manejo de deudas que reemplazó el trabajo en Excel de Tesorería (saldos pendientes ordenados por mora y condiciones de venta, con visibilidad en tiempo real para que los vendedores participen en la regularización de sus clientes); reportería comercial que rescata las ventas directo de la base de datos (facturación, booking de notas de venta pendientes, pendientes de meses anteriores, notas de crédito, bonos y ventas diarias); y una plataforma de ventas y creación de clientes sobre el plugin de e-commerce, con restricciones según las reglas de negocio e ingreso automatizado de parámetros clave para reducir errores y correcciones posteriores.
