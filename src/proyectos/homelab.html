Infraestructura HomeLab

Un notebook que tenía tirado, revivido como servidor propio de producción. Aloja mis proyectos personales en línea —incluido este portafolio— sin exponer la seguridad de mi hogar y sin pagar mensualidad a un PaaS.

Todo empezó con una motivación concreta: quería la comodidad de algo como PythonAnywhere —desplegar proyectos y verlos en línea desde cualquier lado— pero gratis y sin la limitación de estar atado solo a Python con Django o Flask. La solución fue construir mi propio servidor.

Tomé un notebook que tenía sin uso, le instalé Linux y monté Docker gestionado con Portainer. Desde ahí configuro cada proyecto como un stack independiente. Para publicarlos en internet usando mi propia conexión —pero sin exponer la red de mi casa— uso Cloudflare Tunnels, y contraté el dominio tecg8n.xyz (el más barato que encontré) para dar de alta subdominios por proyecto. Con eso puedo levantar y probar proyectos en línea sin importar dónde esté.

Mi ciclo de desarrollo

El HomeLab es la última pieza de un flujo que repito en casi todos mis proyectos:

1 · Desarrollo

VS Code para construir el proyecto, con un ambiente de desarrollo local para iterar rápido antes de publicar.

2 · Control de versiones

GitHub como control de versiones y respaldo: cada cambio queda commiteado y pusheado antes de llegar a producción.

3 · Producción

El servidor personal con Docker: cada proyecto es un stack en Portainer que se despliega y queda en línea a través de Cloudflare.

Corriendo ahora en el HomeLab

El servidor no es un experimento apagado: aloja varios proyectos personales en producción.

  • Portafolio — este mismo sitio.
  • Satelite — asistente IA conectado por API a Gemini que lee mis correos de trabajo.
  • Nexo — plataforma central que expone rápidamente todas mis aplicaciones.
  • Panel IA — generación de imágenes self-hosted con ComfyUI.
  • Master — la instancia de Portainer que gobierna todo lo anterior.
